安全分析概览

应用安全检测

主要对应用进行，业务安全、组件安全、WebView安全性的检测，发现软件登录、支付不合理的设计， 以及暴露 的组件漏洞，输入监听、界面劫持和被 调试等风险的检测。

  • Activity安全 {% if e_act %} 危险 {% else %} 安全 {% endif %}
    导出的组件可以被第三方app任意调用，导致敏感信息泄露或者恶意攻击者精心构造攻击载荷达到攻击的目的。
  • Receiver安全 {% if e_bro %} 危险 {% else %} 安全 {% endif %}
    导出的Receiver组件可造成信息泄露，拒绝服务攻击等。
  • Service安全 {% if e_ser %} 危险 {% else %} 安全 {% endif %}
    Service存在的漏洞主要包括：权限提升、拒绝服务攻击。没有声明任何权限的应用即可在没有任何提示的情况下启动该服务，如果此服务被设置为导出，其他应用便可通过 使用此服务进行提权等操作
  • ContentProvider安全 {% if e_cnt %} 危险 {% else %} 安全 {% endif %}
    Content Provider会导致local sql注入、文件遍历等漏洞，造成应用用户的信息泄露。
  • Intent安全 手工检测
    itent scheme URLs可以通过解析特定格式的URL直接向系统发送意图，导致自身的未导出组件被调用，隐私信息泄露。Itent隐式调用发送的意图可能被第三方劫持，如果发送的 Itent包含隐私信息，则会导致内部隐私数据的泄露。
  • WebView安全 {% if webview == True %} 危险 {% else %} 安全 {% endif %}
    利用android的webView组件中的addJavascriptInterface接口函数，可以实现本地java与js之间交互，但是在安卓4.2以下的系统中，这种方案却给我们的应用带来了很大的安全风险。攻击者如果在页面执行一些非法的JS（诱导用户打开一些钓鱼网站以进 入风险页面），通过远程代码执行，反弹拿到用户手机的shell权限。接下来攻击者就可以在后台默默安装木马，完全控制用户的手机。另外，android webview 组件包含3个隐藏的系统接口：“accessibility”、“accessibilityTraversal”以及“searchBoxJavaBridge_”，同样会造成远程代码执行。
  • 模拟器检测 手工检测
    模拟器具有经济成本低、高度可定制、易于开发、容易部署等优点，攻击者可以通过自己修改定制特定的模拟器来达到监控应用关键函数、获取应用敏感数据，破解应用的目的。为了增加破解阻力，应用中应该进行模拟器检测，添加反模拟器运行代码。
  • 登录涉及缺陷，存在被暴力破解风险 手工检测
    用户登录过程中,未对同一用户的登录失败次数做限制,导致存在被暴力破解的风险。
  • 利用业务逻辑缺陷制作短信炸弹 手工检测
    用户在需要短信验证时，代码中存在逻辑设计缺陷，可使攻击者利用此缺陷造成对指定手机号的短信炸弹。

源码安全检测

主要进行，源码是否有签名、完整性的校验，遭 反编译后泄漏、dex和so文件的隐藏加固，以及 硬编码等风险检测。

  • Dex保护 手工检测
    Dex文件未保护则应用可被重打包插入恶意代码。
  • 代码混淆保护 {% if obfus == "False" %} 危险 {% else %} 安全 {% endif %}
    代码中未调用java混淆保护api，代码可直接被反编译为明文。
  • 可调试安全 {% if debug == True %} 危险 {% else %} 安全 {% endif %}
    在AndroidManifest中将Debugable选项设置为true，则攻击者可直接调试应用。
  • 可备份安全 {% if backup == True %} 一般 {% else %} 安全 {% endif %}
    AndroidManifest配置文件中如果未显示的配置allowBackup为false，则可通过备份数据恢复应用，造成用户信息泄露。
  • 硬编码安全 {% if hardcode == True %} 危险 {% else %} 安全 {% endif %}
    源代码中存在硬编码数据，可能造成加密密钥等敏感信息泄露。
  • 代码加密保护 {% if crypto == "False" %} 危险 {% else %} 安全 {% endif %}
    代码中未调用java加密api，代码可直接被反编译为明文。
  • SO保护 手工检测
    Native so文件中实现较为高级的数据加密，反调试等功能，若So文件未保护，则可能被攻击者对应用so文件具体函数功能进行逆向分析。
  • 异常处理 手工检测
    若代码中未对异常进行处理，则容易导致应用崩溃，类似于DDOS。
  • 权限管理 危险
    若未对权限进行严格控制，则容易导致其他应用通过攻击此应用达到权限提升的目的。
  • 动态调试 {% if debug == True %} 危险 {% else %} 安全 {% endif %}
    如果应用设置为可调试，则攻击者可直接通过调试应用而获取到核心解密函数等敏感信息加密方式。

数据安全检测

针对应用本地存储数据以及涉及用户交互的输入输出，进行窃取和非法访问的检测，检测协议上会话是否安全加密、证书验证以及能否被重放攻击，输出信息是否能为渗透破解提供有效数据等检测。

  • 明文传输用户名、密码等敏感信息 手工检测
    用户登录过程中,在与服务器端交互时明文传输用户名、密码或者验证码等,可导致用户敏感信息泄露。
  • Web表单设计缺陷,存在SQL注入漏洞 手工检测
    开发过程中未对特殊字符进行过滤,攻击者可以通过把 SQL 命令插入到 Web 表单提交或者输入域名或者页面请求的查询字符串,最终达到欺骗 服务器执行恶意的 SQL 命令的目的,这样攻击者就能够对内部数据库进 行增删改操作。
  • 不安全的本地存储 手工检测
    安卓开发者使用多种方法将数据存储在安卓应用中,而存储在本地的数据文件如果未加密,易造成敏感信息泄漏。
  • 后台地址泄露，导致服务器可控 手工检测
    在使用 BurpSuite 等工具对应用进行监听的过程中,发现后台服务器地址。对后台服务器进行测试,若后台服务器存在漏洞,则可控制后台服 务器。
  • 传输数据可修改，造成越权访问 手工检测
    登录一般用户，对访问管理员模块时或在访问个人信息模块时对关键信息进行替换,可越权访问管理员应用模块或其他用户的相应敏感信息。
  • 未使用有效的 token 机制,导致可以绕过鉴权 手工检测
    如果被测应用没有使用有效的 token 机制,对登陆响应中的服务器返回 的鉴权信息进行修改,即可绕过服务器鉴权,直接访问系统内部信息。
  • 边道日志信息移除安全 {% if log == True %} 危险 {% else %} 安全 {% endif %}
    当 APP 处理用户或其它数据源输入的数据时,可能会把数据放在不安全 的位置,容易导致边信道被攻击者利用造成信息泄露。开发者在开发应用是可能将重要数据放在日志记录中进行 测试，发布时未删除相应调试日志数据，造成重要数据泄露。
  • 关键页面存在钓鱼劫持风险，导致用户敏感信息泄露 手工检测
    导出的组件可以被第三方app任意调用，如果对关键页面未做较强的权限管理，则容易被钓鱼劫持。
  • 使用调试证书发布应用 {% if issued == "good" %} 安全 {% else %} 危险 {% endif %}
    使用调试证书发布应用，可导致应用被攻击者进行动态调试，泄露敏感信息及实现核心功能方法。
